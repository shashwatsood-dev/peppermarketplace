Marketplace Procurement Suite

Internal operations & finance intelligence tool for managing freelance creators, work requisitions, and profit margins at a content marketing company.

Tech Stack: React 18TypeScriptViteTailwind CSSshadcn/uiRechartsReact Router v6Tanstack Query

Design: Clean B2B SaaS dashboard — off-white background (#fafafe), white cards, 1px borders (#ebebf0), primary brand #5B5FC7, DM Sans + JetBrains Mono, 13px body, compact density, no gradients, no playful styling.

4. Requisitions /requisitions

Summary Cards Row

Total, In Progress, Pending RMG, Closed, Flagged — each clickable to filter the table or open flags dialog.

Pending Approvals Queue (Admin only)

Toggle button reveals queue of requisitions with status "RMG approval Pending" or taEditedPendingApproval = true. Each row has quick Approve/Reject buttons.

Filters

  • Search (by client name or req ID)
  • Status multi-select dropdown
  • Flow filter (All, Sales, Studio, Freelancer)
  • POD filter

Requisition Table

Grouped by POD. Columns: ID, Flow, Client, Deal ID, Stage (inline editable), Urgency, Revenue, Cost, Margin%, Status (inline editable), Days Open, Flag indicator, Actions (Review, Assign, Update, Edit).

Dialogs

  • Review: RMG notes, approve/reject with rejection reason
  • Assign: POD Lead, Recruiter (with add-new option), Target Date
  • Daily Update: Cumulative funnel metrics (Identified → Selected), blockers, notes. Saves daily log entry
  • Edit: Status, Stage, POD Lead, Recruiter, Target Date, Revenue, Cost, Deal ID, RMG Notes. TA-edited financial changes auto-flag for approval
  • Flags Dialog: Shows red/yellow flagged requisitions based on TAT breach

11. Data Model Summary

Requisition (AdvancedRequisition)

Central entity. Two flows: Sales (pre-deal sample profiles) and Hiring (studio or freelancer). Contains:

  • Identity: ID (S-001, F-001, CS-001), flow type, raised-by info
  • Flow-specific data: SalesFlowData or HiringFlowData (with VSDLineItem[])
  • Workflow: status, RMG notes, rejection reason, POD lead, recruiter, target closure date
  • Financials: total revenue, total cost, gross margin, margin %, target margin %, currency
  • Tracking: daily updates (funnel metrics), audit log (field-change history), TA-edit flag
  • Links: LinkedIn Recruiter link, ATS sheet link

POD → Client → Deal → Creator Hierarchy

EntityKey Fields
PodV2name (Integrated, India B2B, US B2B, FMCG, BFSI), clients[]
ClientV2clientName, vsdName, principalBOPM, seniorBOPM, juniorBOPM, deals[]
DealV2dealName, dealType, totalContractValue, totalCreatorCost, grossMargin%, status, currency, signingEntity, geography, creators[]
DeployedCreatorV2creatorName, role, source, payModel, payRate, totalCost, clientBilling, grossMargin%, dealStatus, capabilityLeadRating, bopmRating, hrbpConnects[], city

Handover

Links a requisition to a creator delivery. Includes: creator info, payment model, finalized pay, currency, share method (email/slack), recruiter, margin tracking.

Flag Logic

Red flag = TAT breached (target date passed). Yellow flag = within 2 days (freelancer/sales) or 5 days (studio) of target.

13. Reproduction Prompt

Build an internal B2B SaaS application called "Marketplace Procurement Suite" using React 18, TypeScript, Vite, Tailwind CSS, and shadcn/ui. It is an operations and finance intelligence tool for a content marketing company that manages freelance creators, work requisitions, and profit margins. DESIGN: Clean, high-density enterprise dashboard. Off-white background (#fafafe), white cards with 14px radius and 1px borders (#ebebf0), primary brand color #5B5FC7. Typography: DM Sans for text, JetBrains Mono for numbers. 13px body, 9-10px uppercase labels, compact spacing. 228px fixed sidebar with 1px right border. 54px topbar. No gradients, no playful styling, no marketing aesthetics. The UI should feel like an enterprise financial workflow system. AUTH: In-memory authentication with three roles — Admin (full access), Pod Lead/Recruiter (Dashboard, Requisitions, Handover, Talent X Client, Studio Dashboard — but Studio hides overall financial summary), Capability Lead/AM (Requisitions own only, Talent X Client own clients only). Admin has a role-switcher dropdown in the sidebar footer. Login page with email/password. Settings page has user management (add users with role assignment). REQUISITIONS: Dual flow system — "Sample Profile Request" (sales/pre-deal) and "Hiring Request" (studio or freelancer). Sales flow captures client, opportunity, deal type, resource types (multi-chip select), currency, expected pay/billing/margin, opportunity stage (9 stages from Pre-Proposal to SLA signed), urgency (Low/Medium/High/Critical). Hiring flow adds Deal ID, studio type, signing entity, POD, client details, deal-level financials (MRR with currency, contract duration, target margin), line items with bidirectional pricing (client price ↔ supply pay linked via margin), replacement hiring toggle. Supports 7 currencies (INR, USD, GBP, EUR, AED, SGD, AUD). Requisition list has: multi-status filter, POD grouping, inline stage/status editing, pending approvals queue for Admin, daily update funnel tracking (profiles identified→selected), full audit log, TAT-based red/yellow flagging, and comprehensive edit dialog with TA-edit approval flow. DASHBOARD: Three tabs — Team Metrics (clickable stat cards opening drill-down dialogs, recruiter comparison table, pipeline bar chart, MoM table), HRBP (connects tracking), Creator Summary (role distribution chart, status grid). Time range selector: 30/60/90/180/365/Overall. TALENT X CLIENT VIEW: POD-based hierarchy (5 PODs: Integrated, India B2B, US B2B, FMCG, BFSI) → Clients → Deals → Creators. Full CRUD on all levels. Creator health ratings (green/yellow/red) with mandatory reason for yellow/red + optional email feedback. Creator statuses: Active/Inactive/Removed/Flagged. Show handover cross-references per deal. STUDIO DASHBOARD: Financial summary cards (role-gated), MoM trend chart, deal cards with creator tables showing HRBP connects, monthly payment tracking, agreement uploads. Geography view groups creators by city. CREATOR HANDOVER: Form linking requisitions to creator deliveries. Multiple creators per handover. Auto-fill margin from requisition. Share via Email/Slack with formatted message copied to clipboard. SETTINGS: User management (Admin adds users with roles, first-login password setup). Custom requisition status management. DATA: Use in-memory stores with mock data. POD hierarchy with 6 clients, 7 deals, 16 creators across 5 PODs. 8 mock requisitions across all three flows. All financial calculations are live (margin auto-calc on cost/revenue changes).
